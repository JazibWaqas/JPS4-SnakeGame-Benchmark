Problem & Approach

A* has a symmetry problem

In a grid, there are thousands of paths to the same goal with the same cost. A* blindly explores all of them. JPS4 doesn't.

🌊

Results — 100×100 Grid

JPS4 checks 70% fewer tiles than A*

Average tiles checked per query · 10% obstacle density
Dijkstra 8,987 tiles
A* 7,903 tiles
JPS4 2,379 tiles ✓
✓ All three algorithms found the exact same shortest path on every single trial. JPS4 is faster — not less accurate.
Bigger map = bigger JPS4 lead
On 50×50: JPS4 does ~29% of A*'s work. On 100×100: same ratio. The gap grows with grid area because JPS4 can "jump" longer distances.
Python hides some of the time savings
JPS4 does more math per jump. In Python, time saved is ~40%. In C++ or compiled code, savings would match the tile count reduction (~70%).
Key Takeaway
JPS4 finds the same perfect path as A*, but by only stopping at corners — skipping everything in between.

The result: 70% less work with zero loss in accuracy.
⚠️ Python overhead limits raw time savings
⚠️ JPS4 is designed for uniform-cost grids only
